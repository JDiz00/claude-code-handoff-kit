Start Here · 5 Minutes

Set up your first handoff

Two ways to use this kit: paste the prompt (zero setup, works anywhere) or install the skill (30 seconds, then it's just /handoff forever). Both end the same way: tomorrow's session resumes without you re-explaining the project.

01

Fastest path: the paste prompt

No install. Try the whole loop once before committing to anything.

Open PROMPT.md from this kit, copy the block inside it, and paste it at the end of any Claude Code session. It writes your handoff and prints a one-line resume command. Next session, paste that line. Done.

If that loop saves you time, install the skill so it's one command instead of a paste.

02

Install the skill (30 seconds)

One folder copy. Run this from the unzipped kit folder.

First, check for collisions. If you already have a /handoff command or skill, this install would shadow it:

ls ~/.claude/skills/ ~/.claude/commands/ 2>/dev/null | grep -i handoff

Nothing printed? You're clear. Install:

mkdir -p ~/.claude/skills && cp -r skill/handoff ~/.claude/skills/

If that skills folder didn't exist before (this is your first skill), restart Claude Code once so it's picked up. Otherwise no restart needed. No config edits, nothing else touched.

Windows note: the commands on this page are for macOS, Linux, WSL, and Git Bash. On native Windows (PowerShell), install with the command below; the skill still works, it just gathers the git state itself instead of using the bundled script. Details in docs/COMPATIBILITY.md.
New-Item -ItemType Directory -Force "$env:USERPROFILE\.claude\skills" ; Copy-Item -Recurse skill\handoff "$env:USERPROFILE\.claude\skills\handoff"
03

End a session

When you're wrapping up, or before a long session gets summarized:

/handoff

The skill will sweep for unresolved mistakes (it asks before fixing anything), write a lean checkpoint (target: ~30 lines) to ~/.claude/handoffs/, and print your resume line. If a handoff already exists for the project, it keeps one .bak of the old version.

04

Resume next session

Open a fresh session in the same project and paste the printed line, e.g.:

Read ~/.claude/handoffs/my-project.md and continue

The fresh session checks your repo's live branch, commit, and dirty counts against the handoff's header before touching any files. If someone (including you, in another window) changed the repo since the handoff was written, it reports the mismatch and stops instead of working from stale info. Then it restates Goal / State / Next in 3 lines and starts the next step.

Running two windows on the same repo at the same time? That's a job for git worktrees, not handoffs. The tripwire catches collisions, it can't prevent them.
05

Good habits (the short version)

  • Hand off at the end of any session you'll continue later, even "in an hour".
  • Never let the ~30-line target trim the Tried/constraints section. Dead ends are the part of your session the repo can't give back.
  • Pointers, not pastes. The handoff records paths and line refs, never file contents.
  • No secrets, tokens, credentials, or customer data in a handoff. Ever.
  • Full rules in docs/POLICY.md, one page.
06

Uninstall

Clean removal, nothing else is affected:

rm -rf ~/.claude/skills/handoff

PowerShell:

Remove-Item -Recurse -Force "$env:USERPROFILE\.claude\skills\handoff"

Your saved handoffs stay in ~/.claude/handoffs/ until you delete them yourself.

End with /handoff. Resume with one line. Stop re-explaining your project.
